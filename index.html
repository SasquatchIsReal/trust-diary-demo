üèóÔ∏è Architecture

This demo showcases pure peer-to-peer communication between a Go service and browser clients. Discovery happens via Nostr relays (decentralized), and the actual data connection is direct WebRTC. No servers, no intermediaries - just cryptographic trust and P2P networking.

Key Components:

  • Nostr Protocol: Decentralized relay network for publishing/discovering WebRTC offers
  • WebRTC DataChannels: Direct browser-to-browser communication after handshake
  • Ed25519 Signatures: Cryptographic identity and message authentication
  • NaCl Box (Optional): Additional end-to-end encryption layer for maximum security
